Overview

What this set covers

Revised after board review. Nav structure, ordering, and the open questions are now locked per the board's direction.

  • Top level (no section label): Conference room, New Task, Dashboard, Inbox.
  • Work (exact order): Tasks, Routines, Goals, Workspaces, Artifacts, Projects.
  • Conference room — new placeholder top-level item, chat-bubble icon, empty page for now (screen 1).
  • New Issue → New Task — renamed; standard nav label color (screen 1).
  • Issues → Tasks — renamed globally across the frontend (nav item, page, all references).
  • Artifacts — new Work item; Projects — new Work item at the bottom of Work, opens the projects list view (screens 1 & 2).
  • Removed — the old collapsible PROJECTS section (per-project rows in the sidebar) is deleted (screen 1).
  • Project tiles — neutral gray rounded rectangle + folder icon; color-only customization this phase, folder icon fixed (screens 2 & 5).
  • Agents section — active agents only when any are running; otherwise up to 5 recently-active + "See all agents" → agents list view with per-row key actions (screens 3 & 4).

4.Agents section — active vs. recently-active

How the sidebar Agents section chooses what to show. Desktop-only diagnostic.

04-agents-states.svg1280×800
Rule: If any agent has a live run → show only those (state A). If none are running → show up to 5 recently-active agents sorted by last heartbeat, plus "See all" (state B). "See all" always opens the full agents list view (screen 3).
Screen 5 · Projects (detail)

5.Project tile — customize icon & color

Default neutral tile and the color picker opened by clicking it. Color-only this phase; the folder icon is fixed. Desktop-only overlay.

05-project-customize.svg1280×800
Why this changes: Per board direction, build color only first — every tile keeps a folder icon (custom-icon picker deferred). Default tiles are neutral gray. Since the per-project sidebar list is removed, tiles live in the Projects list rows and the project detail header, so the color control attaches to the project itself. project.color already exists — no new field needed this phase.
Board answers

Resolved decisions

  • Terminology — Issues → Tasks, renamed globally on the frontend.
  • Conference room — Placeholder only, empty page for now. Confirmed.
  • Projects: dual presence — No. Delete the existing collapsible PROJECTS section; projects are reached only via the Projects list view.
  • Row data — Yes, add a lightweight task-count + budget lookup for the list rows.
  • Project icon — Color only first. Keep a fixed folder icon for every tile; defer custom-icon picker.
  • Work order — Tasks · Routines · Goals · Workspaces · Artifacts · Projects.
Nesting confirmed: Projects (nav) → projects list in the main pane → row → project detail. "See all agents" → agents list in the main pane → row → agent detail.
Wireframes follow the in-tree wireframe skill house style — black on white, 1.5pt strokes, 8px grid, type sizes 12/14/20/28. Red dashed annotations are the call-out layer only and not part of the proposed UI. Nothing here is built yet; this is for plan approval on PAP-53.
